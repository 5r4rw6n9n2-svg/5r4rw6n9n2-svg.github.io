# Обслуживание сайта shelamov.ru — рецепты правок

Практические инструкции по типовым задачам. Общая архитектура — в `README.md`.

> **Главное правило.** Есть два уровня:
> - **Быстрая правка** — отредактировать готовый `.html` в корне и запушить.
>   Просто, но пропадёт при следующем запуске `tools/build.py`.
> - **Правильная правка** — изменить генератор/конфиг и пересобрать (`build.py`).
>   Переживает пересборку. Требует локального `tools/cache/` (см. README).
>
> Для разовых мелочей хватает быстрой правки. Для всего, что может повторяться, —
> правильной.

После любой правки:
```bash
python3 tools/build.py        # если меняли через генератор
python3 -m http.server 8765   # посмотреть локально на http://localhost:8765/
git add -A && git commit -m "…" && git push
```

---

## 1. Поправить текст / опечатку в стихотворении

**Быстро:** открыть `st/<сборник>/<стих>/index.html`, найти нужную строку
`<span class="l" …>…</span>` и исправить текст. Запушить.

Отступ «лесенки» задаётся атрибутами `style="--i:N" data-i="N"` (N — число
символов сдвига). Точное выравнивание потом наводит `assets/poem-align.js`.

> Один и тот же стих может быть в нескольких сборниках — поправьте во всех копиях
> (или правьте через исходник в `tools/cache/` и пересоберите).

---

## 2. Добавить песню (mp3) к стихотворению

1. Положить файл в `music/` с простым именем латиницей, напр. `music/pesnya.mp3`.
2. В `tools/build.py` в словарь `AUDIO` добавить строку:
   ```python
   AUDIO = {
       "za-klukvoj": "vstali-do-rassveta.mp3",
       "o-materi":   "kvas-varganili.mp3",
       "<slug-стиха>": "pesnya.mp3",   # ← новая
   }
   ```
   `<slug-стиха>` — это часть URL: для `/st/samoe-dorogoe/za-klukvoj/` slug = `za-klukvoj`.
   Песня появится на всех копиях стиха во всех сборниках.
3. `python3 tools/build.py`, проверить, запушить.

---

## 3. Добавить / заменить электронную книгу (.ibooks)

1. Положить файл в `books/`, напр. `books/Novaya_kniga.ibooks`.
2. В `tools/build.py` в список `BOOKS` добавить кортеж `(файл, «Название»)`:
   ```python
   BOOKS = [
       ("Naotcheystorone.ibooks", "На отчей стороне"),
       ("Samoe_dorogoe.ibooks",   "Самое дорогое"),
       ("Strofarii.ibooks",       "Строфарии"),
       ("Novaya_kniga.ibooks",    "Новая книга"),   # ← новая
   ]
   ```
3. Пересобрать, запушить. Размер файла подставляется автоматически.

---

## 4. Поправить / добавить ссылку «полный текст» под цитатой (статьи об авторе)

Цитаты в статьях Сытина/Кузнецовой автоматически связываются с нашими стихами.
Если какая-то цитата не нашлась (искажение текста / OCR), добавьте её вручную в
`MANUAL_LINKS` в `tools/build.py`:

```python
MANUAL_LINKS = {
    "а походах щеки впалые": ("na-otchey-storone", "v-more"),
    "<первая строка цитаты в нижнем регистре, ё→е, только буквы и пробелы>":
        ("<сборник>", "<slug стиха>"),
}
```
Ключ — **нормализованная первая строка цитаты**: всё в нижний регистр, `ё`→`е`,
оставить только русские буквы и пробелы (знаки/кавычки убрать).

Пересобрать, запушить.

## 5. Посвящение / эпиграф (курсив справа)

Посвящения определяются автоматически — это строки, которые в исходнике
(`tools/cache/`) были **курсивом** (`<em>`). Короткие → курсив справа
(`.dedication`), длинные примечания → отдельный курсивный блок (`.epigraph`).

- Если посвящение **не распозналось** (его не было курсивом в оригинале) —
  проще всего в готовом `st/.../index.html` обернуть строку:
  замените `<div class="poem"><span class="l">Имя</span>…` — вынесите строку
  посвящения в `<div class="dedication"><span>Имя</span></div>` **перед**
  `<div class="poem">…`.
- Правка «на века» — пометить строку курсивом в исходнике в `tools/cache/<…>.html`
  (обернуть в `<em>…</em>`) и пересобрать.

## 6. Изменить дизайн (цвета, шрифты, отступы)

Всё оформление — в `assets/style.css`. Ключевые переменные в начале файла (`:root`):
```css
--paper   фон страницы (кремовый)
--ink     основной текст
--accent  акцент (кирпично-красный)
--maxw    ширина колонки чтения
--serif   основной шрифт
```
CSS правится напрямую (он не генерируется). Запушить — применится сразу.
Блоки помечены комментариями: «Шапка», «Стихотворение», «Поиск», «Песня» и т. д.

## 7. Добавить новое стихотворение

Сложнее всего, т. к. структура завязана на исходник. Варианты:

**А. Через готовый HTML (быстро).**
1. Скопировать папку существующего стиха, напр.
   `st/na-otchey-storone/abramcevo/` → `st/na-otchey-storone/<новый-slug>/`.
2. В `index.html` заменить заголовок (`<h1 class="page-title">`, `<title>`,
   canonical, breadcrumbs) и строки стиха (`<span class="l">…`).
3. Поправить prev/next у соседних стихов вручную (ссылки `.prev`/`.next`).
4. Добавить стих в `tools/struct.json` (в нужный сборник, в нужное место порядка) —
   чтобы он попал в оглавление и в поиск при следующей пересборке.

**Б. Через исходник (правильно).**
1. Создать страницу-снапшот в `tools/cache/st__<сборник>__<slug>.html` по образцу
   существующей (заголовок в `<h1>`, строки в `<p class="CDt4Ke zfr3Q">`,
   отступы — ведущими пробелами, посвящение — `<em>`).
2. Добавить slug в `tools/struct.json`.
3. `python3 tools/build.py` — страница, prev/next, оглавление, поиск обновятся сами.

---

## 8. Поиск

Поиск работает на `search-index.json`, который **генерируется** `build.py` из всех
стихов и статей. Отдельно править индекс не нужно — он пересобирается. Логика
поиска — в `assets/search.js` (подсветка, сниппет, сортировка: совпадения в
заголовке выше).
