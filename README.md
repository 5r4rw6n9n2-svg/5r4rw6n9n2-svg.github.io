# Сайт Николая Шеламова — shelamov.ru

Статический сайт поэта **Николая Шеламова**, размещён на **GitHub Pages**.
Перенесён с Google Sites: **170 стихотворений** в 4 сборниках, разделы
«Об авторе» (Александр Сытин, Лариса Кузнецова) и «Электронные книги».

- **Адрес:** https://shelamov.ru (и https://www.shelamov.ru → редирект на апекс)
- **Репозиторий / хостинг:** GitHub Pages, репозиторий `5r4rw6n9n2-svg/5r4rw6n9n2-svg.github.io`
- **Деплой:** любой коммит в ветку `main` автоматически пересобирает сайт
- **HTTPS:** включён (Let's Encrypt, сертификат для `shelamov.ru`)

## Как устроен сайт

Сайт **полностью статический** — это готовые HTML-файлы, отдаваемые GitHub Pages.
Сервера нет, поэтому всё динамическое (поиск) работает в браузере.

Есть два способа смотреть на проект:

1. **Готовые HTML в корне** (`index.html`, `st/…`, `ob-avtore/…` и т. д.) — это то,
   что реально видят посетители. Их можно править руками.
2. **Генератор `tools/build.py`** — собирает все эти HTML из снапшота старого сайта
   (`tools/cache/`) и конфигурации. Используется для первичной генерации и
   массовых изменений (дизайн, новые поля, перестройка).

> ⚠️ **Важно:** если запустить `tools/build.py`, он **перезапишет** все
> сгенерированные HTML. Мелкие правки можно делать прямо в HTML, но они пропадут
> при следующей пересборке. Для «вечных» правок меняйте генератор/исходник
> (см. `MAINTENANCE.md`).

## Возможности и как они реализованы

| Возможность | Где | Как |
|---|---|---|
| **Классический дизайн** | `assets/style.css` | Кремовый фон, антиква (serif), книжная типографика, адаптив |
| **«Лесенка» строк** | `poem-align.js` + `--i`/`data-i` в `.l` | Каждая сдвинутая строка ставится ровно после конца предыдущей (+1 символ); скрипт измеряет реальную ширину, работает и для многоступенчатых лесенок. CSS-отступ в `ch` — запасной вариант |
| **Центрирование стиха** | `.poem` в CSS | Блок по центру, текст внутри по левому краю (ширина = по самой длинной строке) |
| **Листание prev/next** | `build.py`, `struct.json` | Порядок стихов берётся из `struct.json` |
| **Посвящения / эпиграфы** | `<em>` в исходнике → `.dedication`/`.epigraph` | Курсивные строки оригинала: короткие → курсив справа; длинные примечания → курсив отдельным блоком |
| **Дефисы → тире** | `ru_dashes()` в `build.py` | `« - »`, `« – »` (с пробелами) и тире в начале/конце строки → `«—»`; внутрисловные дефисы (по-русски, что-то) не трогаются |
| **Цитаты в статьях** | `render_about()` | `<strong>` → заголовок, `<em>`/«ёлочки»/короткие строки → цитата стиха; для цитат ищется наш стих и ставится ссылка «полный текст» |
| **Поиск по сайту** | `search-index.json` + `search.js` + `/poisk/` | Клиентский: индекс грузится в браузер, ищет по заголовкам и тексту, подсветка + сниппет |
| **Аудио (песни)** | `AUDIO` в `build.py`, `/music/` | На страницах стихов плеер `<audio>` с подписью |
| **Электронные книги** | `BOOKS` в `build.py`, `/books/` | Карточки .ibooks с иконкой и кнопкой скачивания |
| **SEO** | `build.py` | `title`/`description`/canonical/OpenGraph/JSON-LD на каждой странице, `sitemap.xml`, `robots.txt`, человекочитаемые URL (сохранены 1:1 со старым сайтом) |

### Конфигурация (в начале `tools/build.py`)
- `DOMAIN`, `SITE`, `AUTHOR`, `TAGLINE` — базовые данные.
- `COLLECTIONS` — сборники: `(slug, «Название»)`, порядок в меню/оглавлении.
- `BOOKS` — `(файл.ibooks, «Название»)` для страницы книг.
- `AUDIO` — `{slug_стиха: «файл.mp3»}` — на какие стихи добавить песню.
- `MANUAL_LINKS` — `{нормализованная_первая_строка_цитаты: (сборник, slug)}` —
  ручная привязка цитат, которые не находятся автоматически (искажения OCR).
- `NAV` — пункты верхнего меню.

## Сборка из исходника (build.py)

Требуется Python 3 и пакеты `beautifulsoup4`, `requests`:

```bash
pip3 install beautifulsoup4 requests
```

Сборка (перегенерирует все HTML из `tools/cache/` и конфигурации):

```bash
python3 tools/build.py
```

Локальный просмотр перед публикацией:

```bash
python3 -m http.server 8765   # затем открыть http://localhost:8765/
```

`tools/crawl.py` — скачивает исходные страницы в `tools/cache/`. Нужен только для
повторного получения исходника (см. предупреждение ниже — старый источник
больше недоступен по домену).

У регистратора `shelamov.ru` настроено:

- **Апекс `shelamov.ru` → A-записи GitHub Pages:**
  `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
- **`www` → CNAME:** `5r4rw6n9n2-svg.github.io`
- Кастомный домен в репозитории задаётся файлом `CNAME` (= `shelamov.ru`).
- В Settings → Pages включён **Enforce HTTPS** (сертификат Let's Encrypt).

## Важные нюансы

- **`tools/cache/` (62 МБ) — только локально** (в `.gitignore`). Это снапшот старого
  сайта на Google Sites — единственная копия оригинала, т. к. домен уже переехал
  на GitHub и старый источник по адресу недоступен. **Не удаляйте без бэкапа** —
  без него нельзя перегенерировать сайт через `build.py` с нуля.
- **`build.py` перезаписывает `CNAME`** при каждом запуске (пишет `shelamov.ru`).
  Это нормально — домен сохраняется.
- **Git push через `gh`:** если обычный `git push` даёт 403, выполните
  `gh auth setup-git` (нужен helper `gh auth git-credential`).
- **Сертификат HTTPS:** при смене домена выпуск может «зависнуть». Лечится
  переотвязкой/повторной привязкой домена:
  `gh api -X PUT .../pages -f cname=` затем `-f cname=shelamov.ru`
  (краткий простой, пока Pages пересобирается), потом `-F https_enforced=true`.
- **Дубли стихов:** один и тот же стих может быть в нескольких сборниках (разные
  URL) — это так и в оригинале. Поиск показывает все копии с пометкой сборника.
